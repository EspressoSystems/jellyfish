pub fn key_gen_v5( ikm: &[u8], salt: &[u8], key_info: &[u8] ) -> Result<(<Self as SignatureScheme>::SigningKey, <Self as SignatureScheme>::VerificationKey), PrimitivesError>

Alternative deterministic key_gen compatible with IRTF draft v5.

  • Secret byte string ikm MUST be infeasible to guess, ideally generated by a trusted source of randomness. ikm MUST be at least 32 bytes long, but it MAY be longer.
  • salt should either be empty or an unstructured byte string. It is RECOMMENDED to fix a uniformly random byte string of length 32. See details here.
  • key_info is optional, it MAY be used to derived multiple independent keys from the same ikm. By default, key_info is the empty string.
