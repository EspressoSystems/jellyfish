Non constant time hash to curve algorithm (a.k.a “hash-and-pray”) The hashing algorithm consists of the following steps:

  1. Hash the bytes to a field element x.
  2. Compute Y = x^3 + 3. (Note: the equation of the BN curve is y^2=x^3+3) 3. Check if Y is a quadratic residue (QR), in which case return y=sqrt(Y) otherwise try with x+1, x+2 etc… until Y is a QR.
  3. Return P=(x,y)

In the future we may switch to a constant time algorithm such as Fouque-Tibouchi https://www.di.ens.fr/~fouque/pub/latincrypt12.pdf

  • H - parameterizable hash function (e.g. SHA256, Keccak)
  • msg - input message
  • returns - A group element in G1
